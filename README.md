# Expense Tracker PM Portfolio

## Overview

This repository contains project management documentation created for the Expense Tracker App project. The repository demonstrates practical application of Product Management and Project Management concepts including requirements gathering, sprint planning, risk management, stakeholder communication, status reporting, change management, and project documentation control.

The objective of this repository is to maintain a structured and version-controlled collection of project artifacts throughout the project lifecycle.

## Folder Description

### BRD

Contains Business Requirement Documents defining business goals, stakeholder requirements, assumptions, and constraints.

### PRD

Contains Product Requirement Documents including features, user stories, acceptance criteria, and product scope.

### Sprint_Planning

Sprint planning guides, sprint goals, capacity planning exercises, and sprint backlog documentation.

### Standups

Daily standup logs, blocker tracking, and sprint execution records.

### Sprint_Reviews

Sprint review agendas, stakeholder feedback records, and increment review documents.

### Retrospectives

Sprint retrospective documents, improvement actions, and continuous improvement tracking.

### MoM

Minutes of Meeting documents including kickoff meetings, status meetings, and escalation meetings.

### Project_Plans

Project plans, Work Breakdown Structures (WBS), milestones, assumptions, constraints, and dependency documentation.

### Risk_Management

Risk Registers, RAID Logs, mitigation plans, and project risk tracking documents.

### Change_Requests

Change Request templates, impact assessments, approvals, and scope management records.

### Status_Reports

Project status reports, RAG status tracking, milestone progress reports, and stakeholder updates.

### AI_PM_Tools

AI-assisted PM artifacts including prompt libraries, AI-generated documentation examples, and AI productivity resources.

## Document Versioning Standard

All documents follow a versioning convention:

### Major Version

Example:

```text
v2.0
```

Used when significant changes are made to scope, structure, requirements, or approved content.

### Minor Version

Example:

```text
v1.1
```

Used for corrections, formatting updates, clarifications, and small revisions.

### Naming Convention

```text
DocumentName_v1.0.docx
```

Examples:

```text
PRD_ExpenseTracker_v1.0.docx
Risk_Register_v1.0.docx
SprintReview_Sprint1_v1.0.docx
```
